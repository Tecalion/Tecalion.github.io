智能物料管理

2024/9/22

校园智能物料管理系统

一、使用工具

​ 前端(Vscode)：Vite + Vue3 + Vue Router + Pinia + Element-Plus + Node.js

​ 后端(Intellij)：SpringBoot3 + Sa-Token + JDBC + Druid + Redis +Spring Security

​ 版本管理：Git

二、现阶段成果

​ 目前仍在开发中，因为是校方私有项目，暂时不便展示。

​ 可查看运行截图确认部分效果。

三、项目概述

​ *该项目库虽然由我创建，但老师说明该项目是校内私有项目，不开源，于是设为私有，如果需要大概浏览可以让我个人展示。

1.项目概述

​ 在读学校的在建项目，预计年底或明年初落地，主要用于协调本校各单位（如项目部、材料部、建筑部等）的工作流程，方便各单位员工查看所需信息与任务。

2.人员分工

​ 指导老师搭建项目大体架构，我与另一名本科大三同学负责不同页面的具体前后端实现，前端vue工作主要是设计页面结构、通过axios连接后端对应api以正确展示列表数据。后端java工作主要是优化sql语句和方法，通过redis缓存发送给前端的列表数据。

3.后续开发

​ 由于原本同组的三位大三同学因项目难度偏高或日常事务较多退组，组内新拉入了两名研究生成员，后续还会持续完善优化，加速开发尽快落地。

四、该项目对我的意义

​ 在这个项目中，我和另一位大三的同学负责与甲方（本校建筑部）沟通需求和跟进开发进度。这是我首次接触代码量最大、形式最正式的项目。尽管遇到了许多新知识，但我们一直在开发过程中不断学习和进步。

​ 前期的主要困难在于不熟悉Vue前端项目与后端SpringBoot项目的分离结构。得益于我在其他项目中的经验，Vue前端界面开发对我来说并不算难事。经过一段时间的学习，我成功掌握了通过Axios库发送HTTP请求与后端进行通信的方法。在后端方面，我也了解了通过Tomcat服务器和SpringBoot的Rest API来接收这些请求。第一个功能模块开发完成后，后续类似模块的开发变得更加顺利。

​ 通过这个项目，我深刻体会到编程设计水平只有在实际开发中才能得到最大的提升。我希望未来能接触更多类似的大型项目，进一步熟练使用Element-Plus等前端UI库，设计出更加美观和实用的界面。

Ai问答bot

2024/9/22

国唐咨询助手（AI问答bot）

一、使用工具

​ 在线低代码编辑平台 ivx（前身ih5）

二、现阶段成果

​ https://editor.ivx.cn/#11693922

三、项目概述

​ *该项目采用在线低代码编辑器ivx开发，需要注册ivx账号并通过在线页面访问项目。

1.项目概述

​ 校内老师要求开发的小程序项目，因为老师指定需要尝试低代码路线，与开发小组的同学分工尝试了不同的低代码平台，由我负责ivx的开发尝试。

2.项目进度

​ 已经于在线编辑器完成聊天页面的设计，调用了智谱清言（ChatGLM）的api获得问答回复，ai对话bot的功能已经完善。关于ai模型微调的问题（回复内容并不符合需求，如果发送”你好“之类的内容，bot会介绍自己是ChatGLM模型，而非ai模拟律师），因为项目初期与老师的研究生交接项目时，在密钥模式和模型微调上并未及时沟通，低代码平台与纯代码开发有很大不同，我们在暑期没有充足的时间再进行模型微调与项目的进一步优化。

​ 在微信上搜索中磁数智技术服务小程序可以尝试使用，该小程序已有测试发布版本。

3.后续安排

​ 低代码平台在小程序的兼容上有很大的问题，老师查看阶段性结果后说要放弃这条技术路线，但这个ivx项目对我来说仍有一定参考意义，所以我保留了下来。

​ 后续会询问老师的需求，时间还足够的话打算尝试使用uniapp+vue3的纯代码模式重新开发该小程序，或者我会考虑学习使用uniapp开发其他ai对话小程序。

四、该项目对我的意义

​ 本项目是在绿算技术公司实习期间，根据老师的需求负责的一个小程序外包项目，原计划提供给上海某律师事务所使用，但由于技术问题暂时延期。

​ 我与甲方（老师）多次沟通需求，并每周召开一次会议讨论项目进度和优化方法。在尝试满足具体的用户需求过程中，我提升了设计和开发小程序的能力，并在推进小程序上线的过程中熟悉了微信小程序的审核流程，了解了微信公众号平台的运作。

​ 遗憾的是，由于暑期实习时间有限、ivx平台的教学文档不够详细，且缺乏实际开发小程序的前辈指导，这个小程序未能实现甲方要求的所有功能，也未能实际投入使用。我希望进入一个更真实高效的工作环境，学习开发功能齐全、用户体验良好的小程序。

bistumaker

2024/9/21

bistumaker.cn（社团官网）

一、使用工具

​ 前端(Vscode)：Vue3 + Pinia + Element-plus + Vue-Router

​ 后端(Intellij)：SpringBoot3 + Mysql + Mybatis

​ 版本管理：git

二、现阶段成果

​ 成果链接：bistumaker.cn

​ 项目链接：Tecalion/BistuMaker: 后续维护 (github.com)

三、项目概述

​ 因后端主要开发人员不是我，此处作品集仅包含我主要负责的前端部分，开发初期我并不熟悉idea内部的git提交方法，故整合的1.0版本由学长提交，后期维护由我fork到自己的仓库中进行。

​ *网页的内容更新可在后台完成，技术更新经由腾讯云服务器的宝塔面板进行。

1.项目概述

​ 本校创客空间社团的信息官网(bistumaker.cn)，前端采用Vue3 + Pinia + Element-plus + Vue-Router进行开发,后端使用SpringBoot3 + Mysql + Mybatis，由两名学生共同开发，社团指导老师提供资金支持购买服务器与域名。

2.人员分工

​ 初期开发由两人共同完成，大二下的我负责前端的前台页面设计，另一位大三下的学长负责后端以及前端的后台页面设计，与学长商讨调研后，决定购用腾讯云的轻量级服务请与域名，由我完成域名备案与使用宝塔面板完成网站挂靠服务器。

​ 初期版本正式运行并投入使用后，由我负责全栈的bug修复与前后台样式优化。

3.后续安排

​ 目前已经完成一次较大的细节和互动体验优化，后续会持续优化维护该网站，进行内容更新或是技术更新。

四、该项目对我的意义

​ 本项目源于我大学期间一直参与和组织的社团。在这个社团中，我学习并了解了许多拓展视野的知识，如Web搭建、鸿蒙开发、机器学习和共享算力等。对于在编程方面从大学才开始起步的我来说，这些知识尤为珍贵。我非常感激这个社团开源开放的精神和合作互助的氛围。

​ 这是我我初次接触接近现代企业标准的项目，项目整体使用Vue和SpringBoot分别主导前后端的开发工作，通过git拉取和提交代码，其中我负责了大部分的前端开发，学习并实践了Vue的相关知识。

​ 在该项目的前端工程中，我在home页面的StarField组件中采用了Vue 2的选项式API风格。在这个组件的设计过程中，我对Vue生态的理解逐渐深入。随后，在其他页面和其他组件中，我采用了Vue 3的组合式API风格，这使得组件逻辑更加灵活和清晰。通过这个项目的完成，我对Vue的理解和掌握有了显著的提升。这次经历不仅让我熟悉了Vue的不同API风格，还增强了我的代码组织能力和项目管理能力。
